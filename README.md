# 📞 Call Center Performance Dashboard (End-to-End Data Analysis Project)

## 🏢 Project Overview

This project presents a complete **end-to-end data analysis workflow**, starting from raw data exploration (EDA) to building an interactive Power BI dashboard.

The objective is to analyze call center operations, improve performance monitoring, and provide actionable insights for better decision-making.

---

![Call Center Dashboard](Call%20Center%20Analysis.png)


## ❗ Problem Statement

Call centers generate large volumes of operational data, but organizations often lack:

* Clear visibility into performance metrics
* Understanding of customer satisfaction trends
* Insights into agent efficiency
* Proper workforce planning during peak hours

Raw data alone is not useful unless it is **cleaned, analyzed, and transformed into insights**.

---

## 🎯 Business Objectives

* Analyze call center data to uncover key patterns
* Clean and prepare raw data for accurate reporting
* Track important KPIs like CSAT, resolution rate, and abandonment rate
* Evaluate agent-level performance
* Identify peak call hours and busiest days
* Build a dashboard for real-time monitoring

---

## 🔍 Key Business Questions

1. What is the total number of calls received?
2. How efficient is the call handling and resolution process?
3. What is the customer satisfaction level (CSAT)?
4. Which agents are top performers and which need improvement?
5. What are the peak hours and busiest days for calls?
6. How are calls distributed across satisfaction levels?

---

## 🧪 Exploratory Data Analysis (EDA)

Before building the dashboard, an in-depth **EDA (Exploratory Data Analysis)** was performed to understand the dataset.

### Key Steps:

* Examined data structure and column types
* Identified missing values and inconsistencies
* Analyzed distribution of:

  * Call durations
  * Satisfaction levels
  * Agent performance metrics
* Checked for outliers and unusual patterns
* Explored relationships between variables

👉 EDA helped in understanding the **data quality and patterns** before visualization.

---

## 🧹 Data Cleaning & Transformation

The raw dataset was cleaned and transformed to ensure accuracy and consistency.

### Data Cleaning:

* Handled missing values
* Removed duplicate records
* Standardized column formats
* Corrected inconsistent data entries

### Data Transformation:

* Created calculated columns and measures
* Converted time fields into usable formats (Hour, Day)
* Aggregated data for KPI calculations
* Structured data for efficient Power BI modeling

👉 Clean data ensures **reliable insights and better decision-making**

## 📊 Dashboard Components

### 1. 📞 Total Call Volume

* Displays total number of calls handled

---

### 2. 🎯 KPI Metrics

* Customer Satisfaction (CSAT)
* Call Resolution Rate

* Average Speed of Answer

👉 These KPIs measure operational efficiency and service quality

---

### 3. 👨‍💼 Agent Performance Analysis

### 4. 😊 Customer Satisfaction Analysis

  * Very Satisfied
  * Satisfied
  * Neutral
  * Dissatisfied
  * Very Dissatisfied
  * Not Served

---

### 5. Call Volume by Hour

* Identifies peak hours
* Helps optimize shift planning

---

### 6. Call Volume by Day

* Highlights busiest days
* Supports workforce allocation
